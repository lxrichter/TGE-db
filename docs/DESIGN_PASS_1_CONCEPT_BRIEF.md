# Design Pass 1 Concept Brief

This brief accompanies the non-production concept route:

- `/design/pass-1`

The route is a visual review lab for the first design phase. It should be used
before changing the live Dashboard, Markets, Analysis, Map, or app shell.

## Scope

Pass 1 covers:

- App Shell
- Dashboard
- Markets
- Analysis
- Map Explorer
- Brand/UI palette
- Chart palette
- Lifecycle status palette
- Governance/review palette
- Typography hierarchy
- Component direction
- Mobile behavior

## Product Direction

The design direction should make the platform feel like a geothermal
intelligence operating system, not a database administration interface.

The product should feel:

- calm
- premium
- evidence-backed
- analytical
- editorially readable
- operationally rigorous
- subscriber-ready

## App Shell Direction

Primary navigation should move toward a left sidebar.

The top utility bar should hold:

- global search
- saved views
- export
- notifications
- AI assistant access
- user/profile context

The top bar should not carry primary product navigation in the final design.

## Dashboard Direction

Dashboard should emphasize:

- Operating MWe
- Pipeline MWe
- market coverage
- evidence coverage
- regional drilldowns
- recent intelligence signals

Operational governance should remain visible but secondary.

## Markets Direction

Markets should use TGE regions as the primary intelligence taxonomy.

World Bank regions remain secondary and should not visually compete with TGE
regions.

Markets should prioritize:

- regional comparison
- country rankings
- operating vs pipeline capacity
- source gaps
- drilldown paths into region and country pages

## Analysis Direction

Analysis should feel closer to consulting-grade intelligence output than a
database report.

Analysis modules should preserve:

- methodology transparency
- attribution logic
- MWe and MWth hierarchy
- chart/table consistency
- internal Governance QA where relevant

## Map Explorer Direction

Map Explorer should be treated as a signature experience.

The map should dominate the interface.

Filters should be:

- secondary
- collapsible
- scalable
- grouped into core and advanced layers

Future map behavior should support:

- expanded map mode
- mobile filter drawer
- record popups as bottom sheets on mobile
- clean screenshot or presentation mode

## Acceptance Signal

Pass 1 is ready to move into implementation when the concept route confirms:

- the platform reads as intelligence-first
- the map feels dominant
- KPI hierarchy is clear
- colors have stable semantic meaning
- mobile hierarchy is credible
- the shell can support subscriber, researcher, editor, and admin use
